Invaders Java Game

Previous Next

This project involved developing a game using the Java language and creating user-interactivity with Arduino controls, this was a combination of hardware and software development. We used an Arduino board in conjunction with a breadboard and Arduino software to allow the user to control the on-screen game with physical controls other than a mouse and keyboard (buttons and a servo).

'Invaders' is first person shooter where you contol a space-tank and must shoot down enough alien spaceships to meet the required score. If you meet the score, you move to the next wave where the aliens are faster, more accuarate and bombs do more damage. There is a random shield that pops up to boost health, this is especially needed in the later waves.


Javascript Website

Previous Next

Learning Arithmetic, is a website made with HTML, Styles and Javascript code. It is a child-friendly website aimed to help kids with learning basic arithmetic. The functionality of the website is, random basic maths questions (multiplation, addition and subtraction) are generated, the user input is then checked on click of the 'check my answer' button where a pop up will appear giving a random message from a list of messages depending on the whether the question was answered correctly (i.e correct answer responses, incorrect answer responses).

E.g a Correct answer would generate a positive encouraging message like 'Correct - good effort'. The amount of correct and incorrect answers are recorded and a live grade and message based on the grade are generated and displayed at the bottom of the webpage.

The Morse Code Translator does exactly what the name suggests, converts english into more code. The Translator is built with the C language and works by processing a text file with english text inside, firstly it generates text stats based on the file such as word counts, line counts etc.

After the stats have been generated the user is prompted to enter the name of the IMC Output file, that is - an empty text file to paste the morse code translation in to. The translation is also printed on-screen inside the terminal window.


Golden Picker

Previous Next

Golden Picker is a small Java based estimation game. The user must select two out of four total rectangles that they believe to be of golden proportions, out of the four rectangles there will aways be two rectangles that are 'golden' in proportion per set (the golden ratio is a mathematical proportion found in design and nature that is thought to be aesthetic).

They choose their two rectangles and submit their answer which is checked, correct answers result in coin rewards, 50 per correct answer for each set of four (so a maximum of 100 coins per set). There is also the ability to set the difficulty, the golden ratio is 1.618, difficulty increases as the ratio of the regular rectangles becomes closer to 1.618 e.g from 2.3 to 1.78 therefore making it harder to distinguish between regular and golden shapes.

3D Modelling and texturing with Maya

Project Overview

The project required that we integrated a 3d Object into a 2d environment. The first point of action was matching the computer generated camera to the real world camera that took the original image of 2d environment.

Camera Matching

We were supplied a technical back plate with objects of known size to help us match the cameras. By changing the camera into position and varying its focal length we recreated the position of the real camera that took the back plate photo. A technique to achieve this camera matching is to make an object whose size and shape we know and manoeuvre the camera until the angle and orientation fit.

Ground Plane and Technical Plate

We then had to create a ground plane that would catch the light of our artificial CG light and the shadow of our 3d object, this was aided by another technical back plate that had a real chrome and grey sphere to help tweak the lighting and darkness of the shadows.

SkyDome

A sky dome was implemented (a global illumination method) was used to simulate indirect lighting e.g. sunlight and it was adjusted and balanced to match that in the technical back plate, a directional light was also required.

Modelling and applying Shaders

We had to make a model using geometric shapes and tools to sculpt them, like for example creating bezels, once the geometry had been created we had to apply UV’s which is a texture showing if there is image distortion as well as the showing the orientation of the image. Shaders also would resonate the effect of whatever material we wanted our geometry to be, e.g. reflectivity.

Render Layers

Render layers that contained separated components (e.g. a sky dome or a sphere) used to build the image of the geometry were used to help in refining eliminating the need to re-render the 3D scene. For example in the shadow layer we only wanted to render the shadows of the balls, not the actual balls.
